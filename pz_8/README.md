# Практична робота №8

---

## Тезисно що було зроблено

- Дослідження механізму часткового запису (partial write) у неблокуючі канали  
- Аналіз роботи системного виклику `lseek` для позиціонування у файлі  
- Тестування швидкодії алгоритму `qsort` на масивах різного об'єму  
- Вивчення результатів системного виклику `fork()` для ідентифікації процесів  
- Дослідження поведінки системного виклику `write()` при записі у FIFO (іменований канал) без активних читачів та обробка `SIGPIPE`  

---

## Програмні вимоги

- Середовище запуску — термінал Linux (Ubuntu)  
- Компілятор — GCC  

---

## Збірка та запуск

### 1. Частковий запис (Завдання 1)

```bash
./1
Requested: 100000, Written: 65536
```

---

### 2. Позиціонування та читання (Завдання 2)

```bash
./2
Buff contains: 2 3 3 7
```

---

### 3. Аналіз швидкодії qsort (Завдання 3)

```bash
./3
Small (n=100000): 0.023 sec
Large (n=1000000): 0.279 sec
```

---

### 4. Результати fork (Завдання 4)

```bash
./4
(Parent proc) Child ID: 3050
(Child proc) PID: 0
```

---

### 5. Поведінка write() при записі у FIFO (Варіант №1)

```bash
./5
13
32
```

---

## Висновки

У ході роботи було досліджено низькорівневу взаємодію програми з ядром ОС через системні виклики вводу/виводу.  

Було проаналізовано роботу підсистем керування процесами та файлової системи, зокрема механізмів передачі даних через дескриптори.  

Виконання завдань продемонструвало важливість перевірки результатів системних викликів, оскільки вони залежать від стану системних ресурсів (розміру буферів, наявності читачів тощо).  

Це є критичним для написання надійного системного коду, що відповідає стандартам POSIX та коректно обробляє виняткові ситуації, сигнали та часткові операції.
