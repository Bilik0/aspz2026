# Практична робота №6

## Тезисно що було зроблено

-   Реалізація власного динамічного алокатора пам'яті на основі
    системного виклику `sbrk()`
-   Створення підтримки списку вільних блоків для відстеження стану
    пам'яті процесу
-   Реалізація механізму **коалесценції** (об'єднання суміжних вільних
    блоків) для запобігання фрагментації

------------------------------------------------------------------------

# Програмні вимоги

-   **Середовище запуску:** Linux terminal (Ubuntu / QEMU)
-   **Компілятор:** GCC

------------------------------------------------------------------------

# Збірка та запуск

## Власний динамічний алокатор пам'яті (Варіант 1)

Написано програму, яка реалізує **кастомний менеджер пам'яті**.\
Замість стандартного `malloc` використовується пряме звернення до ядра
через `sbrk()` для зміни розміру сегмента даних (**heap**).

Для оптимізації роботи зі звільненою пам'яттю реалізовано алгоритм
**коалесценції**.

Під час виконання програми виділяється три блоки пам'яті. При звільненні
центрального блоку об'єднання не відбувається, оскільки сусідні блоки ще
зайняті.\
Коли звільняється перший блок, програма виявляє, що наступний за ним
також вільний, і успішно зливає їх.\
Після звільнення останнього блоку всі фрагменти об'єднуються в один
суцільний вільний масив.

# Висновки

У роботі було розібрано **низькорівневі принципи керування динамічною
пам'яттю в Linux**.\
Створено працюючий прототип алокатора у просторі користувача
(**user-space**), який самостійно контролює виділення пам'яті через
системні виклики.

Практично доведено критичну важливість механізму **коалесценції** при
розробці системного програмного забезпечення.\
Без об'єднання вільних суміжних блоків пам'ять фрагментується, що
призводить до неефективного використання ресурсів операційної системи.

Реалізований алгоритм успішно усуває фрагментацію, зливаючи вивільнені
ділянки в єдиний суцільний простір, готовий для нових алокацій.
