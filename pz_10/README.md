# Практична робота №10

**Виконав:** Володимир Дрогобицький, група ТВ-41

## Тезисно що було зроблено

- Дослідження системного виклику **fork()** для створення нових процесів.
- Аналіз механізму копіювання адресного простору та ізоляції пам'яті.
- Перевірка незалежності глобальних змінних у батьківського та дочірнього процесів.
- Вивчення порядку виконання процесів та використання затримок для синхронізації виводу.
- Експериментальне підтвердження відсутності спільних даних після розгалуження.

## Програмні вимоги

- Середовище запуску — термінал Linux (Ubuntu/QEMU).
- Компілятор — **GCC**.

## Збірка та запуск

### 1. Перевірка ізоляції глобальної змінної (Варіант 1)
Програма ініціалізує глобальну змінну `global_var = 10`. Після виклику `fork()`, батьківський процес змінює її на 20, тоді як дочірній виводить її значення після невеликої затримки. Це дозволяє переконатися, що процеси працюють у різних адресних просторах:

```bash
gcc var_1.c -o var_1
./var_1 
Parent process: global_var = 20
Child process: global_var = 10
```
### Висновки
У ході виконання роботи було детально розглянуто роботу системного виклику fork(). Було підтверджено, що створення нового процесу передбачає створення дубліката батьківського процесу, включаючи всі змінні та стан пам'яті на момент виклику.

Практичний результат (зміна змінної в одному процесі не вплинула на інший) доводить, що Linux використовує механізм повної ізоляції даних. На низькому рівні це реалізується через стратегію Copy-on-Write (CoW), де пам'ять фактично копіюється лише в момент запису. Це забезпечує безпеку та стабільність системи, оскільки процеси не можуть випадково пошкодити дані один одного без використання спеціальних засобів міжпроцесової взаємодії (IPC).
