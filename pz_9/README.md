# Практична робота №9

## Тезисно що було зроблено

- [cite_start]Дослідження облікових записів користувачів у системі через **getent**.
- [cite_start]Отримання адміністративного доступу до захищених файлів **shadow**.
- [cite_start]Аналіз впливу прав власності на можливість редагування та видалення файлів.
- [cite_start]Перевірка прав доступу до системних директорій та бінарних файлів.
- [cite_start]Порівняння поведінки прав доступу на різних файлових системах: **ext4** та **tmpfs**.

## Програмні вимоги

- [cite_start]Середовище запуску — термінал Linux (Ubuntu).
- [cite_start]Компілятор — **GCC**.
- [cite_start]Права користувача — **sudo**.

## Збірка та запуск

### 1. Перегляд облікових записів 
Програма читає базу даних користувачів і знаходить записи з UID вище заданого порогу. [cite_start]Це дозволяє ідентифікувати звичайних та системних користувачів, окрім поточного:
```
./1 1000
Found: nobody (UID: 65534)
```
### 2. Доступ до файлу shadow 
Демонстрація отримання доступу до найбільш захищеного файлу системи, де зберігаються хеші паролів. Виконання вимагає введення пароля адміністратора всередині програми:  


```
user@user:~/lab9$ ./2
[sudo] password for user: 
root:*:20305:0:99999:7:::
daemon:*:20305:0:99999:7:::
```
### 3. Копіювання та видалення 
Досліджено ситуацію, коли файл належить root, але лежить у домашній папці користувача. Було підтверджено, що право на видалення залежить від прав на директорію, а не на сам файл:  

```
user@user:~/lab9$ ./3
Creating file as user...
Copying to home as root...
User trying to edit...
Edit denied
User trying to delete...
rm: remove write-protected regular file '/home/user/copy.txt'? y
Deleted successfully (dir perms allowed it)
```
### 4. Перевірка ідентифікаторів 
Програма виконує системну ідентифікацію, виводячи ім'я поточного користувача та детальний список усіх груп, до яких він належить (включно з групою sudo):  

```
user@user:~/lab9$ ./4
Running whoami:
user

Running id:
uid=1000(user) gid=1000(user) groups=1000(user),4(adm),24(cdrom),27(sudo)
```
### 5. Тестування прав 640 
Було створено файл, власником якого став root, а групою — sudo. Результат показав, що користувач може читати файл через групові права, але не має дозволу на запис у нього:  

```
user@user:~/lab9$ ./5
Creating file...
Root taking ownership, perms 640
Testing read for user:
Hello
Read OK
Testing write for user:
Write Failed
```
### 6. Сканування прав системних файлів
Програма проаналізувала доступність критичних файлів ОС. Встановлено, що пересічний користувач може читати паролі (passwd), запускати утиліти (ls), але повністю заблокований у доступі до shadow:  

```
user@user:~/lab9$ ./6
Checking: /etc/shadow
-R -W -X
Checking: /etc/passwd
+R -W -X
Checking: /usr/bin/ls
+R -W +X
```
### 7. Зміна файлу не власником
Досліджено ситуацію, коли користувач може редагувати файл, не будучи його власником. Протестовано три різні сценарії: надання прав доступу на запис для групи (Group), для всіх користувачів (Others) та використання списків контролю доступу (ACL). В усіх трьох випадках механізми дозволили успішно внести зміни до файлу в обхід обмежень власника:


```
user@user:~/lab9$ ./variant1
Сценарій 1 (Права групи): Успішно змінено.
Сценарій 2 (Права для всіх): Успішно змінено.
Сценарій 3 (ACL): Успішно змінено.
```
### Висновки
У роботі було розібрано систему безпеки та розмежування прав доступу в Linux. З точки зору системного програмування було продемонстровано роботу облікових записів із файлами через системні виклики та стандартні утиліти.  

Було проаналізовано підсистему керування користувачами та правами доступу, яка є критичною для стабільності системи. На практиці показано, як працюють загальні правила безпеки стандарту POSIX: система чітко розрізняє дозволи для власника, групи та інших користувачів. Це запобігає несанкціонованому доступу до конфіденційних даних (наприклад, /etc/shadow).  

Окрему увагу приділено механізму видалення файлів. З'ясувалося, що можливість видалення залежить від прав доступу до каталогу, а не до самого файлу. Це підтверджує ієрархічну перевагу прав директорії над її вмістом у Linux.
