# Практична робота №7  
## Дослідження, моделювання та нестандартні підходи до аналізу процесів, файлових систем, безпеки та ресурсів в Linux

---

## Тезисно що було зроблено

- Використання системного виклику `popen()` для передачі виводу команди `rwho` до утиліти `more` через програмний канал.  
- Імітація роботи системних утиліт `ls -l`, `grep` та `more` за допомогою низькорівневих функцій C.  
- Реалізація рекурсивного обходу дерева директорій та аналізу вкладених об'єктів.  
- Керування правами доступу до файлів через `chmod` та їх видаленням за допомогою `unlink`.  
- Вимірювання часу виконання коду в мілісекундах та генерація випадкових чисел у заданих діапазонах.  
- Визначення структури ієрархії директорій для користувача з виключенням циклічних посилань (Варіант №1).  

---

## Програмні вимоги

- Середовище запуску — термінал Linux (Ubuntu)  
- Компілятор — GCC (обов’язковий прапорець `-Wall`)  
- Системні утиліти — `rwho`, `more`, `grep`

## Висновки

Під час виконання роботи було розібрано принципи взаємодії програми з файловою системою та процесами в Linux.  

На практиці продемонстровано роботу з системними структурами даних (`stat`) для отримання метаданих файлів та каталогів.  

Реалізація аналогів системних утиліт (`ls`, `grep`, `more`) дозволила зрозуміти логіку їх функціонування на рівні системних викликів та обробки файлових дескрипторів.  

Окрему увагу було приділено безпечному обходу ієрархії папок, що дозволяє будувати коректні дерева директорій без ризику зациклення через символьні посилання.
